# 중첩클래스와 중첩인터페이스

### 1. 중첩 클래스 란?

클래스 내부에 선언된 또 다른 클래스입니다.

- 목적: 두 클래스가 서로 긴밀한 관계일 때, 외부 클래스의 멤버에 쉽게 접근하고 코드의 복잡성을 줄이기(캡슐화) 위해 사용한다.
- 크게 **멤버 클래스**와 **로컬 클래스**로 나뉩니다.

- 멤버클래스:
    - 인스턴스 클래스 : 클래스 내부 선언, static 키워드 없이 선언. 바깥 객체가 있어야 생성 가능.
    - 정적 멤버클래스 : 클래스 내부 선언,static 키워드 붙임. 바깥 객체 없이 바로 생성 가능.
- 로컬클래스 :
    - 메소드 내부 선언, 메소드 안에서만 사용하고 메소드 종료 시 소멸.

### 2. 익명 객체 (Anonymous Object)

이 장의 하이라이트 중 하나입니다. **이름이 없는 객체**를 의미하며, 단발성으로 사용될 때 유용.

- **익명 자식 객체:** 부모 클래스를 상속받아 생성 (주로 메소드 재정의 목적)
- **익명 구현 객체:** 인터페이스를 구현해서 생성 (이벤트 처리, 스레드 등에서 자주 사용)

### 3. 중첩 인터페이스

클래스의 멤버로 선언된 인터페이스입니다. 해당 클래스와 긴밀한 관계를 맺는 구현 객체를 만들기 위해 사용됩니다. (예: 버튼 클래스 내부의 ClickListener 인터페이스)
