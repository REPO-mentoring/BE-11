# 예외처리

## 1. 예외(Exception)와 에러(Error)

- **에러(Error):** 하드웨어 고장이나 JVM 메모리 부족(OOM) 등 프로그램 코드만으로는 해결할 수 없는 심각한 오류
- **예외(Exception):** 사용자의 잘못된 조작이나 개발자의 코딩 실수로 발생하는 오류입니다. 예외 처리를 통해 프로그램을 계속 실행 상태로 유지할 수 있다.

## 2. 예외의 종류

자바의 모든 예외 클래스는 java.lang.Exception 클래스를 상속받는다.

일반 예외 (Checked Exception): 컴파일 단계에서 체크가 된다. 반드시 try-catch나 throw가 필요하다. classNotFoundException, IOException

실행 예외(RuntimeException) :Runtime에서 발생한다 개발자의 판단에 따라 처리 한다. NullPointerException, ArrayIndexOutOfBoundsException

## 3. 예외 처리 코드 (try-catch-finally)

예외가 발생했을 때 프로그램이 멈추지 않도록 하는 기본 구조입니다.

```java
try {
    // 예외 발생 가능성이 있는 코드
} catch (예외클래스 e) {
    // 예외가 발생했을 때 실행할 코드
} finally {
    // 예외 발생 여부와 상관없이 무조건 실행되는 코드 (주로 리소스 해제)
}
```

## 4. 예외 떠넘기기 (throws)

메소드 내부에서 예외를 처리하지 않고, 자신을 호출한 곳으로 예외를 떠넘기는 방법입니다.

- 리턴타입 메소드명(매개변수) throws 예외클래스1, 예외클래스2 { ... }
- 나중에 이 메소드를 호출하는 곳에서 반드시 try-catch로 예외 처리를 해주어야 합니다.

## 5. 리소스 자동 닫기 (try-with-resources)

Java 7부터 도입된 기능으로, InputStream, Connection 등 사용 후 반드시 닫아야(close) 하는 리소스를 자동으로 관리해 줍니다.

- java.lang.AutoCloseable 인터페이스를 구현한 클래스만 사용 가능합니다.
- finally에서 명시적으로 close()를 호출할 필요가 없어 코드가 깔끔해집니다.

## 6. 사용자 정의 예외와 예외 발생시키기

자바 표준 API에서 제공하지 않는 예외를 직접 만들어 사용할 수 있습니다.

- **정의:** Exception이나 RuntimeException을 상속받아 클래스를 만듭니다.
- **발생:** throw new 내가만든예외("메시지"); 키워드를 사용하여 강제로 예외를 던집니다.
