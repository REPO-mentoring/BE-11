# 인터페이스

### 인터페이스의 개념

- 인터페이스는 객체의 사용 방법을 정의한 타입입니다.
- 역할 : 두 객체를 연결하는 중계 역할을 합니다. (예: 사람 → 리모콘(인터페이스) → TV/오디오)
- 목적: 개발 코드를 수정하지 않고, 사용하는 객체만 교체함으로써 실행 결과를 다양하게 만들기 위함이다
- 선언 : class 대신 interface키워드로 선언

```java
public interface 인터페이스명{...}
```

---

### 인터페이스 구성 멤버

인터페이스는 생성자가 없다

- 상수필드
    - 인터페이스의 필드는 무조건 public static final 특성을 갖습니다. (생략해도 자동 컴파일이 된다)
    - 값을 바꿀 수 없는 상수입니다.
- 추상 메소드
    - 리턴타입, 매개변수, 메소드 명만 기재후 {}블럭이 없음
    - 무조건 public abstract 특성을 갖습니다.
    - 구현 클래스(자식)는 이 메소드들을 반드시 Override해야 합니다.
- 디포르 메소드 :
    - default 키워드가 붙으며, 인터페이스 내부에서 {}실행블럭을 가진다
    - 기존 인터페이스를 확장해서 새로운 기능을 추가할 때, 기존 구현 클래스들이 에러 나지 않게 하기 위해 도입됨
- 정적 메소드
    - static 키워드가 붙으며, 객체 없이 인터페이스 이름으로 바로 호출 가능한 메소드이다.

---

### 구현클래스

인터페이스에 정의된 추상 메소드를 실제로 코드로 작성하는 클래스입니다.

- **선언:** public class 구현클래스 implement 인터페이스명{ … }
- **규칙:** 인터페이스의 모든 추상 메소드를 재정의해야 하며, 재정의 시 접근 제한자는 반드시 public 이어야 합니다.
- **인터페이스 변수:** 구현 객체는 인터페이스 타입 변수에 대입되어 사용됩니다.
    - 인터페이스 변수 = new 구현객체();

---

### 4. 다중 상속과 다중 구현

자바에서 클래스는 다중 상속이 불가능하지만, 인터페이스는 가능합니다.

- **다중 구현:** 클래스는 여러 인터페이스를 동시에 구현할 수 있습니다.
- **인터페이스 상속:** 인터페이스끼리는 extends를 통해 상속받을 수 있으며, **다중 상속**도 가능합니다.

---

### 5. 다형성

7장의 상속과 원리는 같지만, 사용 빈도는 인터페이스가 훨씬 높습니다.

- **필드의 다형성:** 자동차의 타이어를 winterTire에서 summerTire로 갈아 끼워도, Tire 인터페이스 규격만 지켰다면 자동차는 문제없이 굴러갑니다.
- **매개변수의 다형성:** 메소드 파라미터로 인터페이스 타입을 받으면, 그 인터페이스를 구현한 **어떤 객체든** 들어올 수 있습니다.
- void drive(Vehicle vehicle) { … }-> 버스, 택시, 트럭 모두 전달 가능

---

### 6. 강제 타입 변환과 instanceof

상속과 마찬가지로, 인터페이스 타입으로 변환된 객체를 다시 원래의 구현 클래스 타입으로 되돌릴 때 사용합니다.

- **Casting:** Bus bus = (Bus)vehicle
- **Instanceof:** if(vehicle instanceof Bus) { … } (안전하게 확인 후 변환)
