# 상속

### 1. 상속이란?

상속은 이미 잘 개발된 클래스(부모)를 재사용해서 새로운 클래스(자식)를 만드는 것입니다.

```java
class 자식클래스 extends 부모클래스 { ... }
```

자식 클래스는 부모의 필드와 메소드를 모두 물려받습니다. (단, private 멤버와 다른 패키지의 default 멤버는 제외한다)

### 2. 부모 생성자 호출(super)

super() : 자식 생성자의 맨 첫 줄에는 컴파일러가 자동으로 super();를 추가합니다. super()는 부모의 기본 생성자를 호출합니다.
주의할 점: 만약 부모 클래스에 기본 생성자가 없고 매개변수가 있는 생성자만 있다면, 자식 생성자에서 반드시 명시적으로 super(매개변수)를 호출해줘야 에러가 나지 않습니다.

### 3. 오버라이딩

부모에게 물려받은 메소드가 자식 클래스에 맞지 않을 때, 자식 클래스에서 내용을 고쳐서 사용하는 것입니다.

- 규칙
    - 부모의 메소드 이름, 리턴타입, 매개변수 리스트가 모두 같아야한다
    - 접근제한자를 더 좁은 범위로 좁힐 수 없다.(public → private)
- @Override : 메소드 위에 이 어노테이션을 붙히면 컴파일러가 제대로 재정의 하였는지 검사해준다.
- super.메소드() : 재정의를 하였더라도 부모의 원래 메소드를 호출하고 싶을 때 사용

### 4. final 클래스와 final 키워드

- final은 “최종의”라는 뜻으로 상속에서는 변경금지를 뜻한다.
- final class : 상속할 수 없는 클래스가 됨
- final method : 오버라이딩할 수 없는 메소드가 됨(자식이 부모의 내용을 수정할 수 없음)

### 5. 다형성

하나의 타입에 여러 객체를 대입해 다양한 동작을 수행하는 성질

- 자동 타입 변환
    - 자식은 부모의 특징을 다 가지고 있으므로, 자식 객체는 부모 타입으로 자동 변환될 수 있습니다.
        - 부모타입 변수 = 자식객체; (가능)
            - **특징:** 부모 클래스에 선언된 필드와 메소드만 접근 가능합니다. 단, **자식에서 오버라이딩된 메소드가 있다면 자식의 메소드가 호출**됩니다. (이것이 다형성의 핵심)
- 강제 타입 변환 (Casting)
    - 부모 타입으로 변환된 객체를 다시 자식 타입으로 되돌리는 것입니다.
        - 자식타입 변수 = (자식타입) 부모타입변수;
            - **조건:** 처음에 자식 객체였다가 부모 타입으로 변환된 경우에만 가능합니다.
- 객체 타입 확인 (instanceof)
    - 강제 타입 변환 전에, 이 변수가 실제 어떤 객체인지 확인할 때 사용합니다.
        - if (parent instanceof Child) { ... }

### 6. 추상 클래스 (Abstract Class)

추상 클래스란 미완성 설계도이다. 공통된 필드와 메소드의 이름을 통일할 목적으로 사용한다.

- abstract 키워드: 클래스 앞에 붙이면 객체를 직접 생성(new)할 수 없습니다. 상속을 통해서만 사용 가능합니다.
- 추상 메소드: 메소드 선언부만 있고 실행 블록 {}이 없는 메소드입니다.
    - public abstract void method
    - 강제성: 자식 클래스는 반드시 추상 메소드를 오버라이딩해서 실행 내용을 채워야 합니다.
